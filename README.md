## Troubleshooting Local Routes

If you see `ERR_CONNECTION_REFUSED` on `localhost:3001` for routes like `/terms-and-conditions`, verify the active dev server port first.

1. Run `npm run dev` and check the `Local:` URL printed by Next.js.
2. Open the app using that exact host and port.
3. If needed, confirm active listeners with PowerShell:

```powershell
Get-NetTCPConnection -LocalPort 3000,3001 -State Listen |
	Select-Object LocalAddress, LocalPort, State |
	Format-Table -AutoSize
```

This project typically runs on `http://localhost:3000` unless the `PORT` environment variable is explicitly changed.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
